SAE1.05/1.06
Projet HTML / CSS

Préparation / Présentation globale

Ce projet avait pour but la création d'un site web institutionnel d'une entreprise du secteur numérique pour les "nuls".

Cette S.A.É. est repartie en 3 parties :
- Définition du besoin (cahier des chargesà).
- Conception (wireframe) du site.
- Réalisation (développement) du site.

Note groupe était consitué de 4 membres.

Le sujet porte sur l'accessibilité de notre site web aux amateurs mais aussi aux moins connaisseurs.

Nous avons dû expliquer une entreprise en simplifiant les termes techniques (lexique) pour rendre l'entreprise accessible à la génération Z.

Contraintes : Cahier des charges

Au début du projet nous avons dû monter un recueille des besoins pour notre site web, réunir des informations sur l'entreprise puis les triées.

Un collègue a été nommé chef d'équipe pour la rédaction de celui-ci.

Phase 2 : Conception / Wireframe

Lors de la phase de conception nous avons produit un design très homogène sur l'entête et le pied de page pour gardé le design synthétique

Ce style de conception permet à l'équipe de travailler que sur le contenue des pages et non aux styles car il est prédéfini.

La sémantique à aussi joué un rôle important dans cette conception, il fallait garder un tout cohérent.

Phase finale : Réalisation [HTML/Css]

Durant cette phase décisive, il a fallu développer cette conception avec les languages HTML/Css.

En suivant les maquettes (wireframe) que nous avons produit lors de la phase de conception et en respectant le cahier des charges.

Nous avons pu produire un site web fonctionnel.

Ressources qui alimentent ce projet

Les ressources utilisées pour ce projet sont :
• R1.02 [Développement d'interfaces web]
Elle comprend nos connaissances dans le langage de programmation Html/Css.

• R1.11 [Bases de la communication]
Elle comprend nos méthodes de communication / coordination lors d'un contexte projet.

Nos/Mon rôle(s)

Nous avons réparti les rôles de la manière suivante :

• Pour ce projet je me suis porter volontaire pour être le chef de projet. Je devais donc prendre du recul sur notre travail et coordonnée tous les membres pour finir les tâches.

• J'étais également chargé de faire le compte rendu à toute l'équipe et de régler les dernières erreurs avant le rendu final.

• Les autres membres avait chacun 2 page à faire pour le site.

J'ai commencé donc en amont à faire l'entête et le pied de page pour laisser plus de temps aux autres membres pour s'occuper du contenu de leur page.
