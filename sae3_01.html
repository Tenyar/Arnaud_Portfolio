SAE3.01
Application web ludo-éducative

Préparation / Présentation globale

Ce projet avait pour objectif de créer, au sein d'une équipe, une application en suivant une démarche de développement itérative ou incrémentale. En partant d'un besoin décrit de manière imprécise, l'objectif est de collecter et formaliser le besoin puis de développer une application communicante intégrant la manipulation des données et respectant les paradigmes de qualité (ergonomie des IHM, qualité logicielle, sécurité…). L'application développée devra être une application client-serveur sécurisée s'appuyant sur une base de données.

Ce/Cette projet/S.A.É est regroupé(e) en trois thèmes :
- Recherche & Veille d'information et du marché
- Conception
- Réalisation

Il s'agissait d'un projet réalisé dans un groupe d'étudiants de 6 à 7 développeurs spécialiser dans le développement d'application web & mobile.
Ainsi que d'un à deux étudiants spécialiser dans le déploiement d'applications communicantes et sécurisé.

Notre serious game ludo-éducatif avait comme thème la musique et comme publique les élèves de 3ème.

Les technologies web choisis pour ce projet sont :

- Le langage Javascript pour le côté client, et les différents jeux proposés.
- Le langage PHP pour le côté serveur, avec la communication avec la base de données et l'API REST.
- Une API REST pour communiquer les données/scores des élèves sur les différents jeux.
- Le framework Vexflow pour la création de partition musicale intéractive.
- La libraire Javascript frontend Pixi.js.
- Sqlite3 pour tester l'application avant l'utilisation de serveur dédiée à une base de données.

Les technologies choisis pour les serveurs utilisé dans ce projet sont :
- Un serveur (machine virtuel) web Apache sous OS linux, distribution Debian 12.
- Un serveur (machine virtuel) de base de données MariaDB.
- Un certificat SSL pour assurer une encryption des données avec le serveur web.
- Raspberry Pi 3th gen, pour rendre accessible notre site web en ligne via l'adresse IP de ma box SFR.
- Paquet SSH pour effectuer une connexion entre le serveur web et le raspberry Pi.

Réalisation

Durant la réalisation, l'équipe de développement web à réaliser 3 mini-jeu, le but étant pour un élève de troisième d'éffectuer cette suite de mini-jeux dans l'ordre et d'obtenir un score dans chacun d'entre eux.
L'enseignant peut ainsi visualiser le score de ses élèves et analyser les résutltats.

De mon côté j'ai mis en place un serveur web (sous Apache) et une base de donnée.
Le serveur web avait un certificat signé (SSL) par IONOS qui garantissais le chiffrement entre les échanges des utilisateurs et notre serveur web.

J'ai aussi mis en place avec l'aide d'une raspberry PI 3 un pont SSH entre le raspberry et le serveur de l'université.
Permettant de prendre l'adresse IP de ma box SFR comme adresse de notre site web et ainsi l'ouvrir sur le réseau public.
